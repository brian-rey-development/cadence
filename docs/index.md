## Product Thesis

Cadence is built on a defensible bet: **constraint is a feature**. The daily 7-task cap, three fixed life areas, and the ritual of closing the day are deliberate limitations that force prioritization.

This is the right call. Every productivity tool that broke through had a strong opinionated point of view. Cadence has one.

The current feature set gets you to "interesting prototype." These docs map the path to a tool people **cannot stop using**.

---

## 90-Day Priority Stack

1. **Ship push notifications** - Daily close reminder at 6pm is the highest-ROI feature in the backlog. Nothing else matters if users forget the app exists.
2. **Onboarding flow** - New users need to experience the value loop within 5 minutes. An empty Today view kills potential.
3. **Tomorrow pre-seeding from daily close** - The close-to-open loop is the core daily retention mechanism.
4. **Goal milestones + health indicators** - Goals without progress tracking are aspirational labels. Milestones make the Goals view worth visiting daily.
5. **Longitudinal mood + energy trend** - Mood data is captured but never surfaced. A 30-day trend makes the product feel like it understands you.
